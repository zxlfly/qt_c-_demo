# 2026-05-03

## CustomTableDelegate 修复
- 问题：`paint()` 中手绘了 QLineEdit/QSpinBox/QComboBox 的外观（写死颜色、边框），导致非编辑状态样式与系统原生不一致。
- 修复：使用 QStyle API 绘制原生控件外观，替代手绘。非编辑状态下看起来和编辑时弹出的真实控件一致。
- 具体改动：
  - QLineEdit(列0,3)：`PE_PanelLineEdit` + `PE_FrameLineEdit` 绘制边框，`SE_LineEditContents` 定位文本
  - QSpinBox(列1)：`CC_SpinBox` 绘制完整控件，`SC_SpinBoxEditField` 定位文本
  - QComboBox(列2)：`CC_ComboBox` 绘制完整控件（含下拉箭头），选中时用 `SC_ComboBoxEditField` 覆盖文本颜色
- 注意：Qt 6 中 `CE_FrameLineEdit` 和 `SE_SpinBoxContents` 已移除，需用 PrimitiveElement 和 subControlRect 替代

## setIndexWidget 重影问题分析与修复
- 问题：`buyong` 表格使用 `setIndexWidget` 放置持久化控件，编辑后出现重影
- 原因：`setIndexWidget` 的 Widget 是叠加层(overlay)，底层默认委托的 `paint()` 仍在绘制文字 → 两层文字重叠
- 修复：添加 `NoPaintDelegate`（在 mainwindow.cpp 中定义），重写 `paint()` 只画选中背景不画文字

## CustomTableDelegate 完整重写
- 用户反馈四类问题：控件样式不对、选中态异常、编辑器不弹出、数据不同步

  - `fillRect(option.rect, option.palette.base())` 覆盖选中高亮
  - SpinBox 补充 `buttonSymbols`、ComboBox 补充 `iconSize` 配置
  - 新增 `sizeHint()` 保证行高 ≥ 24px
  - mainwindow.cpp 为 paint 表格添加 `setEditTriggers(DoubleClicked | SelectedClicked)`
