# 2026-05-03

## CustomTableDelegate 修复
- 问题：`paint()` 中手绘了 QLineEdit/QSpinBox/QComboBox 的外观（写死颜色、边框），导致非编辑状态样式与系统原生不一致。
- 修复：使用 QStyle API 绘制原生控件外观，替代手绘。非编辑状态下看起来和编辑时弹出的真实控件一致。
- 具体改动：
  - QLineEdit(列0,3)：`PE_PanelLineEdit` + `PE_FrameLineEdit` 绘制边框，`SE_LineEditContents` 定位文本
  - QSpinBox(列1)：`CC_SpinBox` 绘制完整控件，`SC_SpinBoxEditField` 定位文本
  - QComboBox(列2)：`CC_ComboBox` 绘制完整控件（含下拉箭头），选中时用 `SC_ComboBoxEditField` 覆盖文本颜色
- 注意：Qt 6 中 `CE_FrameLineEdit` 和 `SE_SpinBoxContents` 已移除，需用 PrimitiveElement 和 subControlRect 替代
- 编译通过（链接时 SYS.exe 被占用导致 Permission denied，关闭运行中的程序即可）
